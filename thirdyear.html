Third Year

Posted by Neil Seward on April 17, 2015

Third year was definitely very tough initially going in. The topics covered were very foreign at the time and, often for the most difficult courses, I was lectured by uninterested profs that had poor teaching methods. Operating systems was particularly hard in that the first half of the course was taught in C. Learning child process creation with a language never really used in your program that has weak debugging methods isn't always the most fun. Languages and Compilers was definitely the hardest, but it was the most rewarding. At the beginning of the class, I had no clue about anything that went on behind compilers and how basic text was converted into language readable objects and then ultimately to machine code. Ultimately at the end, I was able to make a Java based, Pascal arithmetic like compiler that evaluated simple addition, mutliplication, sin comutations along with other variations.

Artificial Intelligence

I loved this course. The professor who taught this course was actually very excited to teach us about the different advanced algorithms used in the field such as A*, Breadth, Best First, and Depth First.

What really intrigued me were the last few chapters taught in this course on machine learning. The professor introduced classification trees and clustering algorithms like K-Nearest Neighbour. Those were some of the most interesting lectures that I've been to so far at school.

LiveTraffic

Repository

Our group designed a really cool live traffic and incident display using two overlapping API for the display and incident data. Google Maps API provided an easy platform to grid the different traffic incidents in the Greater Toronto Area received from the HERE Maps API.

We designed our backend with PHP because it looked more easily learned than Node.js at the moment. Our entire group first experienced server side programming and getting JSON responses from a web service.

The main goal of the project was to fill the database with data gotten from an API and display that information in the form of views. Another requirement was to provide some sort of API service to get JSON responses from the database for outside use.

Our group decided to go beyond the initial requirements and thought it would be cool to show exact areas that have had a large amount of traffic incidents in the past. We implemented this idea by using a simple geo range query to find incidents that have a certain threshold of incidents within a boxed area around them. Because we only got data on incidents over a few days, there was not a noticible amount of correlation in any areas in Toronto.
